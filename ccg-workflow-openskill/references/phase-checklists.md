# Phase Checklists (OpenSkill Gates)

## A. 主流程 6 阶段 Gate

### 1) 研究
- [ ] 目标、范围、非目标明确
- [ ] 约束条件可执行
- [ ] 成功标准可验证
- 输出：`Decisions/Risks/Evidence/Next`

### 2) 构思
- [ ] 至少 2 个方案
- [ ] 每个方案包含优缺点与适用边界
- [ ] 给出单一推荐方案
- 输出：方案对比表 + 推荐结论

### 3) 计划
- [ ] 任务粒度可执行（每项有完成定义）
- [ ] 标注依赖和并行边界
- [ ] 包含验证命令与回滚策略
- 输出：任务清单 + 验证清单

### 4) 执行
- [ ] 仅执行已批准任务
- [ ] 每次变更有对应验证
- [ ] 关键节点有中间产出
- 输出：实现结果 + 验证结果

### 5) 优化
- [ ] 性能热点是否量化
- [ ] 可维护性问题是否收敛
- [ ] 不引入新范围
- 输出：优化前后对比

### 6) 评审
- [ ] 与计划目标逐项对照
- [ ] 回归测试通过
- [ ] 风险剩余项可说明
- 输出：交付结论（可发布/需补做）

## B. Spec 通道 Gate

### spec-init
- [ ] `openspec` 命令可用
- [ ] 项目存在必要目录结构
- [ ] 初始化动作可复现

### spec-research
- [ ] 需求已转为约束
- [ ] 边界案例明确
- [ ] 验收标准可自动化

### spec-plan
- [ ] 关键歧义清零
- [ ] 关键技术参数已定
- [ ] PBT 属性和反例策略已列出

### spec-impl
- [ ] 只做 `tasks` 范围内改动
- [ ] 每个任务有验证证据
- [ ] 失败可回滚

### spec-review
- [ ] Critical 已清零
- [ ] Warning 有处置结论
- [ ] 满足归档门禁

## C. Team 通道 Gate

### team-research
- [ ] 并行探索覆盖关键模块
- [ ] 汇总为统一约束

### team-plan
- [ ] 任务拆分 3-6
- [ ] 文件边界互斥
- [ ] 依赖链清晰

### team-exec
- [ ] 每轮并行结果统一汇总
- [ ] 冲突自动降级串行
- [ ] 关键改动有验证

### team-review
- [ ] 双模型审查均完成
- [ ] 审查结论已分级
- [ ] 最终裁决唯一

## D. 标准汇总模板

```markdown
## 阶段汇总

### Decisions
- 已确定约束 1
- 已确定约束 2

### Risks
- 风险 1 + 触发条件
- 风险 2 + 缓解策略

### Evidence
- 命令/测试: <command>
- 关键文件: <path>

### Next
- 下一阶段入口条件
- 需要用户确认的问题
```
